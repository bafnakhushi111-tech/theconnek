Section 02
Build Quality
Infrastructure, SEO, metadata, and technical compliance. Almost entirely resolved from V1.
Filter:
Done
SEO metadata — complete stack
Title, meta description, keywords, Open Graph (title, description, image, locale, type, url), Twitter card (summary_large_image, site @_the_connek), canonical URL, and robots index/follow — all set in layout.tsx via Next.js Metadata API.
Done
Dynamic OG image at /opengraph-image
Next.js ImageResponse generates a 1200×630 branded card. Plus Jakarta Sans loaded via fetch for the renderer. Used as og:image and twitter:image in all metadata.
Done
JSON-LD Organization schema
Injected via <script type="application/ld+json"> in layout.tsx <head>. Covers name, url, logo, description, sameAs (Instagram), and foundingLocation (India).
Done
GDPR/DPDP compliant analytics setup
GA4 (G-JDBRXGBYBX) and Microsoft Clarity (x1p3rgic1z) only render after explicit cookie acceptance. CookieBanner.tsx reads/writes localStorage. Analytics.tsx listens on storage events for real-time consent changes.
Done
Neon PostgreSQL — no pausing
Replaced Supabase (which paused after 1 week inactivity) with Neon. Separate mentees and mentors tables. Neon client initialized inside the POST handler to avoid build-time env var errors.
Done
Custom 404 page
app/not-found.tsx with on-brand copy ("This page wandered off"), consistent dark theme, and a button back to home. No more Vercel default error screen.
Remaining
No sitemap.xml or robots.txt
Neither file is present. Google has no crawl guidance for the now multi-page site (/about, /faq, /contact, /privacy, /terms). Next.js can auto-generate a sitemap via app/sitemap.ts — under 30 minutes to add, immediate indexing benefit for all pages.
Remaining
Select dropdowns missing focus rings
The role and experience <select> elements use appearance-none but have no custom onFocus border handler — keyboard users see no visible focus state. Text inputs already have this (onFocus sets border to accent colour). Same pattern needs applying to the selects.

Body text colour bumped from #6B7FA3 to #8A9CB8 across all pages and components: homepage, FAQ, contact, cookie banner, not-found, thank-you, and mentors. Noticeably better contrast on dark backgrounds.

Contact page + FAQ page
/contact with hello@theconnek.in and "We reply within 24 hours". /faq with 8 questions covering pricing, eligibility, matching, study circles, and guesstimate practice. Both linked from sidebar and footer.
Open — Key Gap
Stats strip has no real numbers
The three stats currently read "100% Free / No algorithm / Built for India" — all claims, no numbers. A count of real members (even "47 members" or "12 mentors") would be far more convincing than a feature claim. Once real signups exist, replace one stat with a DB count or a hard-coded number.
Minor
Skip-to-content link missing
Keyboard and screen reader users must tab through the full fixed nav before reaching page content. A visually-hidden "Skip to main content" anchor at the top of the <body> is the WCAG standard fix — roughly 5 lines of CSS and one HTML link.
Section 04 · Biggest Gap
Trust & Social Proof
Scored 4/10. This section is what holds the site back from 8.5+. The infrastructure exists — it just needs real people in it.
Filter:
Biggest Gap
Mentor profiles section is empty
The Mentors component exists and will render a full card grid — but the MENTORS array is empty, so the entire section hides itself. A first-time visitor never sees a single real human on the site. Every competitor surveyed (MentorCruise, ADPList, MentorTogether) front-loads real mentor cards. Even 3 real names, companies, and roles would be the single biggest trust lift on the page. Action: send 3–4 names and they go live in minutes.
Biggest Gap
No testimonials or attributed quotes
Not a single person on the site has said "Connek helped me." The value prop cards describe outcomes abstractly — no names, no faces. Once the first real users join, even one honest quote ("I had a 20-minute call with a McKinsey associate through Connek — changed how I thought about recruiting. — Priya, NMIMS '25") beats every line of copy on the page. A static pull-quote section above the footer is a 2-hour build.
Done
About page with founder story
/about exists with a founder narrative explaining why Connek was built. Linked from sidebar and footer. Professional tone without being corporate. CTAs to join and back to home.
Done
Instagram handle visible and linked
@_the_connek linked from footer and sidebar nav. JSON-LD sameAs also references the Instagram profile. Social presence is findable.
Done
Legal pages build institutional trust
Privacy policy and terms of service signal a serious product. Plain-English writing, DPDP Act 2023 + GDPR references, all data processors named. Especially important for Indian users evaluating a new platform.
Done
Dual-persona copy signals genuine two-sidedness
The professional flow has distinct copy ("A 30-minute conversation can change someone's direction"), separate form fields, and its own accent colour. Signals that the supply side is real and considered, not an afterthought.
The one move that changes everything
Add 3–4 real mentor profiles. The component, the DB, and the design are all ready — the array is just empty. Send names, companies, and roles (LinkedIn profiles are fine as a source) and the section goes live immediately. This alone would push the overall score from 7.8 to 8.5+ — more than any other single change left on the list. Combined with a real member count in the stats strip, you're at 8.7.
