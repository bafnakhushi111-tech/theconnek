Section 1: The Internet — How It Works

Before any specific tool, you need to understand the plumbing. Everything else builds on this.

The Browser and the Server

Core Concept
Every website has two parts: a client (your browser — Chrome, Safari) and a server (a computer somewhere in the world that stores and runs your website's code). When you visit theconnek.com, your browser sends a request to Vercel's servers, which send back the HTML/CSS/JavaScript your browser needs to display the page.
Analogy
Think of a restaurant. The client is you (the customer). The server is the kitchen. You place an order (a request), the kitchen prepares it (processes it), and the waiter brings it back (response). You never see the kitchen — you just see the food.

HTTP / HTTPS

What It Is
HTTP stands for HyperText Transfer Protocol. It's the language that browsers and servers use to talk to each other. HTTPS is the secure version (the S = SSL encryption). Every URL starting with https:// means the communication between your browser and the server is encrypted — no one in the middle can read it.
Why it matters for Connek
When a user submits the waitlist form, their name and email travel from their browser to our server over HTTPS. Without it, the data could be intercepted. Vercel enforces HTTPS automatically on all deployments.

GET vs POST (HTTP Methods)

What They Are
HTTP requests have "methods" — verbs that describe what you want to do:
  • GET — "Give me data." Used when you open a webpage. No body, just a request.
  • POST — "Here is data, do something with it." Used when you submit a form. Has a body (the form data).
  • PUT / PATCH — Update existing data.
  • DELETE — Delete something.
Analogy
GET = asking a librarian "what books do you have?"   POST = handing the librarian a new book and saying "please add this."
In Connek
When the user fills the waitlist form and clicks "Join →", the browser sends a POST request to /api/waitlist with the form data in the body (name, email, college, role, user_type as JSON).

JSON

What It Is
JSON stands for JavaScript Object Notation. It's the standard format for sending data between a browser and a server. It looks like this:

{ "name": "Priya Sharma", "email": "priya@example.com", "role": "Consulting" }

It's just structured text — both the browser and server can read it easily.
Analogy
JSON is like a filled-in form. Instead of a paper form with labeled boxes, it's a digital format where each field has a label ("key") and a value.

Section 2: API — The Most Important Concept

What is an API?

Definition
API stands for Application Programming Interface. An API is a defined way for two pieces of software to talk to each other. It's a contract: "if you send me this, I'll do that and send you back this."

You don't need to know how the other software works internally — you just follow the API contract.
Analogy
A restaurant menu is an API. You don't know how the chef makes the pasta. You just order "Pasta Carbonara" (the request), and you get a plate of food (the response). The menu is the contract — it tells you what you can order and what you'll get.

Similarly, Supabase's API says: "Send me a POST request with your table name and data, I'll insert it into the database."

Our API Route — /api/waitlist

What We Built
In Connek, we built our own API endpoint. It lives in app/api/waitlist/route.ts. It does three things when called:
  1. Validates the data (checks all fields are present)
  2. Calls Supabase's API to insert the row into the database
  3. Calls Resend's API to send an email notification
It returns { "success": true } if everything worked, or an error message if not.
User clicks "Join →" ↓ Browser sends POST to /api/waitlist with form data ↓ Our API route (route.ts) runs on Vercel's servers ↓ Calls Supabase API → row inserted into Postgres database ↓ Calls Resend API → email sent to bafnakhushi111@gmail.com ↓ Returns { success: true } to browser ↓ Browser redirects user to /thank-you

REST API

What It Is
REST stands for Representational State Transfer. It's a set of design rules for how APIs should be structured. Most APIs you'll encounter (Supabase, Resend, Stripe, etc.) follow REST. The rules are simple:
  • Use URLs to represent resources: /waitlist represents the waitlist
  • Use HTTP methods to define actions: POST = create, GET = read, DELETE = delete
  • Responses are in JSON
  • Each request is self-contained (no "memory" between requests)
Interview Answer
"Our waitlist endpoint follows REST principles — POST to /api/waitlist creates a new entry. I used Next.js Route Handlers which automatically map the HTTP method to the right function."

Serverless Functions

What It Is
A serverless function is code that runs in the cloud only when called — there's no server sitting idle waiting. Vercel turns every API route in our Next.js app into a serverless function automatically.

When someone hits /api/waitlist, Vercel "spins up" a tiny computer to run that function, it completes, and the computer is gone. You pay only for actual usage, not idle time.
Analogy
Traditional server = hiring a full-time employee who sits at a desk all day waiting for work.
Serverless = hiring a freelancer who you call only when you need them. They show up, do the job, leave. You pay per task, not per hour.
Interview Answer
"I used Next.js API routes which Vercel deploys as serverless functions. This means I have zero server management overhead — the function scales automatically with traffic and I'm not paying for idle compute."

Section 3: The Frontend — What Users See

React

What It Is
React is a JavaScript library made by Meta (Facebook) for building user interfaces. Instead of writing HTML directly, you write "components" — reusable pieces of UI. React re-renders only the parts of the page that actually changed, making it fast.
Analogy
Imagine a physical dashboard with gauges and switches. Traditional HTML = redrawing the entire dashboard every time one needle moves. React = only moving the specific needle that changed. Much faster, much smoother.
Key Concept: Component
A component is a reusable piece of UI with its own logic. In Connek: Logo.tsx is a component (the Connek logo), Footer.tsx is a component (the bottom of every page), Mentors.tsx is a component (the mentors grid). You drop them into any page like Lego bricks: <Footer />
Key Concept: State
State is data that can change and when it changes, React re-renders the relevant parts of the page.

In Connek, userType is state — it holds either "candidate" or "professional". When you click the toggle, userType changes → React re-renders the headline, form, and steps to show the right content. No page reload needed.

Next.js

What It Is
Next.js is a framework built on top of React. A framework = opinionated set of tools and conventions that give you a full application, not just a UI library.

Next.js adds on top of React:
  • File-based routing — folder structure = URL structure
  • API routes — backend code collocated with frontend
  • Server-side rendering — page HTML generated on the server (better SEO)
  • Image optimization, font loading, OG images — built in
  • TypeScript, Tailwind — first-class support
React vs Next.js Analogy
React = engine.  Next.js = the full car (engine + steering + seats + GPS + fuel gauge). You could build the car yourself from just the engine, but why would you?

App Router (Next.js feature)

What It Is
The App Router is how Next.js knows which file to show for which URL. The rule is simple: folder name = URL path.
FileURL it serves
app/page.tsxtheconnek.com/
app/thank-you/page.tsxtheconnek.com/thank-you
app/privacy/page.tsxtheconnek.com/privacy
app/about/page.tsxtheconnek.com/about
app/api/waitlist/route.tstheconnek.com/api/waitlist
app/not-found.tsxAny URL that doesn't exist (404)
No config file, no mapping to set up — just folders and files.

TypeScript

What It Is
TypeScript = JavaScript + type safety. You tell the computer what "type" every variable is (string, number, boolean, etc.) and TypeScript checks your code for mistakes before it runs.

Example:
type UserType = "candidate" | "professional"
This means the variable can only ever be one of those two exact strings. If you accidentally write "candidat" (typo), TypeScript errors immediately — before users ever see it.
Interview Answer
"I used TypeScript across the entire codebase for type safety. It catches errors at compile time — for example, I typed the form state so missing fields or wrong types throw a build error, not a runtime bug in production."

Tailwind CSS

What It Is
Tailwind is a CSS framework. CSS is what makes websites look the way they do — colors, fonts, spacing, layouts. Instead of writing separate CSS files, Tailwind gives you pre-made "utility classes" you apply directly in your HTML/JSX.

Traditional CSS:
.button { background-color: blue; padding: 8px 16px; border-radius: 9999px; }

Tailwind equivalent:
className="bg-blue-500 px-4 py-2 rounded-full"

Same result, no separate file, faster to write and change.

Framer Motion

What It Is
Framer Motion is a React library for animations. Without it, things appear and disappear instantly. With it, you get smooth transitions, fade-ins, and scroll-triggered reveals.

Key things we used it for in Connek:
  • fadeUp — sections fade in from slightly below as you scroll. Signals "there's more content" rather than dumping everything at once.
  • AnimatePresence — when you toggle between candidate/professional, old content fades out and new content fades in. Without this, it would just snap to the new content jarringly.
  • stagger — cards animate in one-by-one instead of all at once. Guides the eye. Makes the page feel alive.
  • whileHover / whileTap — buttons scale slightly when hovered or clicked. Micro-interaction that signals "this is clickable."
Interview Answer
"I used Framer Motion for scroll-triggered animations and the candidate/professional toggle transition. The toggle uses AnimatePresence so content crossfades — it communicates that the page is personalised, not just static text swapped out."

Section 4: The Backend — Data and Logic

Database

What It Is
A database is organised, persistent storage for data. "Persistent" means it survives when the server restarts — unlike variables in code which disappear when the program stops.

Two main types:
  • Relational (SQL) — data in tables with rows and columns. Like Excel, but more powerful and with strict rules. Examples: PostgreSQL, MySQL, SQLite.
  • Non-relational (NoSQL) — data in flexible documents (like JSON). Examples: MongoDB, Firebase Firestore.
We use PostgreSQL (relational).
Analogy
A database is like a filing cabinet. Each "table" is a drawer. Each "row" is a folder in that drawer. Each "column" is a labeled section inside the folder (name, email, date, etc.). You can search, sort, and filter any data instantly.

PostgreSQL

What It Is
PostgreSQL (often called "Postgres") is the world's most popular open-source relational database. "Open-source" means the code is free and maintained by a community.

Our waitlist table looks like this:

2Rahul Mehtarahul@gmail.comMcKinseyStrategyprofessional2026-06-08
Each row = one signup. Each column = one field of data.

Supabase

What It Is
Supabase is a hosted platform that gives you a PostgreSQL database + a REST API to talk to it + a dashboard to view your data — without needing to manage any servers yourself.

Normally, using a database requires:
  • Renting a server (AWS/GCP)
  • Installing PostgreSQL on it
  • Writing connection code
  • Managing security, backups, scaling
Supabase handles all of that. You just sign up, create a table, and call their API from your code.
Analogy
Supabase is to databases what Gmail is to email servers. Running your own email server is possible but complex. Gmail abstracts it — you just log in and send emails. Supabase lets you just "use a database" without managing the infrastructure.
Interview Answer
"I used Supabase as my database layer — it's a hosted PostgreSQL service with a JavaScript client. Each waitlist signup inserts a row via their API. I chose it because it's free for small scale, requires zero server management, and the dashboard lets me see signups in real time."
Important: Free Tier Behaviour
Supabase's free tier pauses the database after 1 week of no activity. If no one signs up for a week, the next person who tries will get an error — the database needs to be "unpaused" manually in the Supabase dashboard. This is a known trade-off of the free tier.

Environment Variables

What They Are
Environment variables are secret values stored outside your code — they're injected at runtime by the platform (Vercel). You reference them as process.env.VARIABLE_NAME in code but never write the actual value in the code file.

Our environment variables:
Variable NameWhat It HoldsWho Uses It
NEXT_PUBLIC_SUPABASE_URLThe URL of our Supabase projectAPI route + browser
NEXT_PUBLIC_SUPABASE_ANON_KEYPublic key for SupabaseAPI route + browser
RESEND_API_KEYSecret key for Resend email serviceAPI route only (server)
NEXT_PUBLIC_GA_IDGoogle Analytics measurement IDBrowser (analytics)
NEXT_PUBLIC_CLARITY_IDMicrosoft Clarity project IDBrowser (analytics)
Variables starting with NEXT_PUBLIC_ are exposed to the browser. Others (like RESEND_API_KEY) are server-only secrets.
Why not just put the keys in the code?
If your API key is in your code and your code is on GitHub, anyone who sees the repo has your API key. They could run up your bill, access your database, or send emails from your account. Environment variables keep secrets out of the codebase entirely.

Section 5: Tools and Services

Git and GitHub

What Git Is
Git is a version control system. It tracks every change you make to your code — what changed, when, and who changed it. You can go back to any previous version at any time.

Key commands:
  • git add — stage files you want to save
  • git commit — save a snapshot of staged changes with a message
  • git push — upload your commits to GitHub (the cloud)
  • git pull — download changes from GitHub to your computer
What GitHub Is
GitHub is a website that hosts Git repositories (codebases) in the cloud. It's where the Connek code lives at github.com/bafnakhushi111-tech/theconnek. It's also how Vercel gets our code — we connect Vercel to GitHub and it auto-deploys every time we push.
Analogy
Git = track changes in Word. GitHub = Google Drive for your code (stores it in the cloud, lets you share it, see the history).

Vercel

What It Is
Vercel is a cloud deployment platform — it takes your Next.js code from GitHub and makes it available on the internet. Every time you do git push, Vercel automatically:
  1. Detects the new code
  2. Builds it (converts TypeScript → JavaScript, bundles assets)
  3. Deploys it globally (copies to servers in multiple regions worldwide)
  4. Routes your domain (theconnek.com) to the new version
Zero manual steps after the first setup.
Analogy
Without Vercel, publishing a website = buying a physical server, renting data centre space, installing software, configuring the network, setting up SSL, managing security patches. With Vercel, you just push code to GitHub.

It's like the difference between owning a printing press vs. clicking "Print" at Staples.
What Vercel Does For Connek
  • Hosts the site at theconnek.com and theconnek.in (custom domains we pointed to Vercel)
  • Stores our environment variables (Supabase keys, Resend key) securely
  • Runs our serverless API routes (/api/waitlist)
  • Handles SSL/HTTPS automatically
  • CDN — serves static files (images, CSS) from servers close to each user globally

Resend

What It Is
Resend is a transactional email API. "Transactional" means event-triggered automated emails — as opposed to marketing bulk emails (Mailchimp). Every time someone joins the waitlist, Resend sends you an email from hello@theconnek.com with the signup details.

Why not Gmail or SMTP directly?
  • Gmail limits you to ~500 emails/day and has deliverability issues
  • Resend handles DKIM and SPF records (email authentication standards that prevent your mail from landing in spam)
  • It sends from your own domain (theconnek.com), not a generic Gmail address
  • Simple API: one function call with To, From, Subject, and HTML body

Framer (vs Framer Motion)

Clarification
Framer = a no-code/low-code website builder (like Webflow). We did NOT use this.
Framer Motion = a React animation library made by the same company. We used this. They are different products despite the similar name.

npm and node_modules

What They Are
npm (Node Package Manager) is the tool that installs external libraries. When we say we "use Framer Motion" — npm downloads the Framer Motion code from the internet and saves it in a folder called node_modules inside the project.

package.json is the file that lists every library the project depends on. Running npm install reads package.json and installs everything listed.
Analogy
npm is like Amazon for code. package.json is your shopping list. npm install places the order. node_modules is the delivery — all the packages sitting in your project folder.

Section 6: SEO and Discovery

SEO (Search Engine Optimisation)

What It Is
SEO is the practice of making your website discoverable by search engines (Google, Bing). When someone Googles "career networking platform India," you want theconnek.com to appear. SEO determines how high you rank.

Key SEO signals:
  • Page title and meta description — what shows in Google results
  • Sitemap — tells Google which pages exist
  • robots.txt — tells Google which pages to crawl or skip
  • Page speed — faster sites rank higher
  • Mobile-friendliness — Google penalises non-mobile sites
  • Content quality — relevant, original content ranks better

sitemap.ts

What It Is
A sitemap is an XML file at /sitemap.xml that lists every page on your website with the date it was last updated. Google reads this to know what to index.

Without a sitemap, Google discovers pages by following links — slow and incomplete. With one, you explicitly hand Google your full page list.

In Next.js, you just export a function from app/sitemap.ts — Next.js generates the XML automatically. No manual file editing needed.

robots.ts

What It Is
/robots.txt is a text file that tells web crawlers (Google's bots) which pages they can and cannot access. We allow all pages and point to our sitemap.

Example: if you had an admin panel at /admin, you'd tell robots NOT to crawl it. A public sitemap URL gets submitted to Google Search Console.

OG Image (Open Graph)

What It Is
When you share a link on WhatsApp, LinkedIn, or X (Twitter), a preview card appears — with an image, title, and description. This preview is driven by "Open Graph" meta tags in the page's HTML.

Next.js has a special convention: put a file called opengraph-image.tsx in your app folder and it auto-generates a PNG image using React components. This image becomes the share card.

Why it matters: links with OG images get significantly more clicks than plain text links. It's the difference between a news article showing a photo vs. a bare URL.
Interview Answer
"I implemented OG images using Next.js's built-in next/og library. The image is generated server-side as a React component — so it includes our logo and tagline and renders as a proper share card on WhatsApp and LinkedIn."

Section 7: Analytics

Google Analytics 4 (GA4)

What It Is
GA4 is Google's free analytics platform. You add a small JavaScript snippet to your website and it tracks:
  • How many people visited (sessions, users)
  • Where they came from (direct, Google search, Instagram link)
  • Which pages they visited and in what order
  • How long they stayed
  • What device/country they're on
  • Custom events (e.g. "clicked Join button", "submitted form")
This data lives in your Google Analytics dashboard — accessible from any browser.

Microsoft Clarity

What It Is
Clarity is Microsoft's free behavioural analytics tool. While GA4 gives you numbers, Clarity gives you:
  • Session recordings — watch a recording of real users clicking, scrolling, and typing on your site
  • Heatmaps — see which parts of the page users click most (shown as a colour heat map overlaid on your site)
  • Scroll maps — see how far down users typically scroll before leaving
This is invaluable for UX decisions. Instead of guessing why users aren't converting, you literally watch what they do.
GA4 vs Clarity
GA4 = the speedometer and fuel gauge of your car (numbers, metrics). Clarity = a dashcam footage of every trip (you can watch what actually happened).

Analytics Scaffold (how we built it)

What We Did
We built an Analytics.tsx component that:
  1. Loads GA4's script if NEXT_PUBLIC_GA_ID env var is set
  2. Loads Clarity's script if NEXT_PUBLIC_CLARITY_ID env var is set
  3. Does nothing if neither is set (so local development isn't polluted with fake analytics)
To turn on analytics, you just add the env vars to Vercel — no code changes needed.

Section 8: Legal Pages

Privacy Policy

What It Is and Why It's Required
A Privacy Policy is a legal document that tells users:
  • What personal data you collect (name, email)
  • Why you collect it (to manage the waitlist)
  • How you store it (Supabase / Postgres)
  • Who you share it with (Resend for email delivery)
  • User rights (how to request deletion)
India's DPDP Act (Digital Personal Data Protection Act, 2023) requires any platform collecting Indian users' personal data to have a Privacy Policy. Without one, you're legally exposed. The EU has GDPR with similar requirements.

Terms of Service

What It Is
Terms of Service is a legal agreement between you and your users. It defines:
  • What users can and cannot do on your platform
  • What you're responsible for (and what you're not)
  • How disputes are resolved
  • Your right to terminate accounts / remove content
Without it, a disgruntled user could claim anything and you'd have no documented agreement to reference.

Section 10: Interview Cheat Sheet

QuestionYour Answer
What did you build?A full-stack community platform — React frontend, serverless API backend, PostgreSQL database, transactional email, SEO, analytics, and legal pages. All deployed on Vercel.
Why Next.js?It gives React + API routes + SSR + deployment in one framework. I didn't need a separate Express server — the API routes are serverless functions that Vercel manages automatically.
How does the waitlist work end-to-end?User submits form → browser POSTs JSON to /api/waitlist → serverless function inserts a row into Supabase Postgres and sends a Resend notification email → returns success → user is redirected to /thank-you.

How do you handle duplicate signups?Supabase has a unique constraint on the email column. On duplicate, it throws error code 23505 (Postgres unique violation). We catch that specifically and return a 409 with a clear message.

What analytics did you set up?GA4 for quantitative metrics (traffic, sources, conversions) and Microsoft Clarity for qualitative behavioural data (session recordings, heatmaps). Both gated on env vars so they're off in development.
What's the tech stack?Next.js 16 (App Router, TypeScript), Tailwind CSS, Framer Motion, Supabase (PostgreSQL), Resend, Vercel. GitHub for version control.
What would you improve?Rate limiting on the API endpoint (prevent spam signups), email verification before storing, and moving off Supabase free tier before scaling (to avoid the 1-week pause issue).
