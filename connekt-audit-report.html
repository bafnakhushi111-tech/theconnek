Section 02
Build Quality
Infrastructure, SEO foundations, performance, and accessibility. All V1 findings remain unresolved.
Filter:
Critical · Unresolved
No og:image / og:description / twitter:card
Every link shared on WhatsApp, LinkedIn, Slack, or X generates a blank preview. For an India-first community product whose entire growth motion is WhatsApp groups and LinkedIn posts among students, this is a fatal omission. Add og:image (1200×630), og:description, twitter:card via Next.js Metadata API.
Critical · Unresolved
No analytics installed
GA4, Hotjar, and Microsoft Clarity all undetected. Zero visibility into conversion rates, scroll depth, drop-off points, traffic sources. Install GA4 via @next/third-parties and Clarity (free heatmapping) before running any outreach.
High · Unresolved
No sitemap.xml or robots.txt
Both return 404. Google has no crawl guidance. Install next-sitemap with a postbuild script — under 45 minutes.
High · Unresolved
No branded 404 page
Every mistyped URL serves Vercel's default error screen. With 13 known 404 routes today and more coming, any deep link = permanent funnel exit. Branded 404 with CTA back to home keeps users in funnel.
High · Unresolved
No structured data (JSON-LD)
No Organization, WebSite, or FAQPage schema. Zero rich-result eligibility in SERPs — no sitelinks, FAQ dropdowns, or enhanced snippets. 1-hour fix with direct SERP CTR impact.
High · Unresolved
Verify SSG, not CSR rendering
Marketing landing page must be SSG, not CSR, for LCP + SEO. Confirm App Router server components or getStaticProps — no getServerSideProps on the homepage.
Medium · Unresolved
Form has no validation, error states, or success copy
No "please enter a valid email", no CAPTCHA, no visible confirmation message in markup. If submission fails, users see nothing. Add inline validation, error messaging, success state before any meaningful volume.
Medium · Unresolved
No lang attribute or ARIA labels
The <html> element appears to be missing lang="en" — WCAG requirement + minor SEO signal. No aria-label or role on form controls. 15-minute fix.

Good
Single H1 with correct hierarchy

Section 05
Tech Stack: Supabase Scalability
Unchanged from V1. Supabase remains the right choice for pre-launch, but the connection-pooling fix is still outstanding — and now more urgent given the India pivot (campus WhatsApp groups can drive sudden spikes).
1
Connection exhaustion — hits at ~50–200 concurrent API requests
Free/micro tier allows only 200 pooler connections via Supavisor. Next.js on Vercel with Prisma opens a new DB connection per Lambda invocation. Moderate Indian campus traffic — a viral WhatsApp share, an IIM-A LinkedIn post — exhausts this in minutes. First wall teams hit; looks like a mysterious outage, not a connection limit.
Fix now: switch all DB connections to Supavisor in transaction mode (not session mode). 1 hour. Buys runway through Small/Medium compute ($20–40/mo).
2
Realtime subscription bottleneck — hits at ~500 concurrent WebSocket users (Pro)
Supabase's Postgres Changes realtime processes events on a single thread for ordering. Compute upgrades don't fix this — it's architectural. Any messaging/presence/live-feed feature with 5K+ simultaneous users will lag.
When building realtime: use Broadcast mode instead of Postgres Changes for high-frequency events. Broadcast bypasses Postgres entirely.
3
Egress cost shock — hits at ~100K MAU (~$630/month)
A realistic 100K MAU app with 5TB/month bandwidth runs ~$630/month on Supabase Pro. Storage at $0.021/GB adds up for media uploads.
Enable spend cap on Pro to prevent surprise bills. Consider Cloudflare R2 ($0.015/GB, no egress) for media/file storage.
4
No horizontal scaling — vertical only
Supabase scales by bigger compute. No read replicas, no multi-region writes, no horizontal sharding. Multi-region or horizontal scale (likely at 500K+ MAU) requires a platform change, not a config update.
Plan the Neon + Clerk migration before you hit this, not after.
⚡
Migration trigger: Start the Supabase → Neon + Clerk conversation when you cross 10K–50K MAU or when monthly Supabase bill exceeds $200–$300/month. Before that point, migration cost outweighs benefit. After, you're paying for problems you could have solved.
Recommendation (unchanged from V1)
Stay on Supabase now — right tool for pre-launch. But fix connection mode to transaction pooling immediately. When monthly bill crosses $200/month or you hit real connection/realtime limits, move to Neon + Clerk: same Postgres paradigm, better autoscaling, stronger auth DX, ~30% lower cost at mid-scale.
